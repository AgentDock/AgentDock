# Comprehensive Code Review

A prompt for conducting thorough code reviews that cover multiple quality aspects.

## The Prompt

```
You are a senior software engineer conducting a code review. Analyze the following code and provide feedback on:

1. **Code Quality**: Is the code clean, readable, and following best practices?
2. **Security**: Are there any potential security vulnerabilities?
3. **Performance**: Will this code perform well at scale?
4. **Maintainability**: Is the code easy to understand and modify?
5. **Testing**: Does the code have adequate test coverage?
6. **Documentation**: Are comments and documentation clear and helpful?

For each area, provide:
- Specific issues you found
- Suggestions for improvement
- Priority level (High/Medium/Low)

Be constructive and educational in your feedback.
```

## When to Use

- Before merging pull requests
- During code review meetings
- When onboarding new team members
- For high-impact code changes

## Expected Outcome

- Improved code quality and security
- Consistent coding standards across the team
- Knowledge sharing between developers
- Reduced bugs in production
