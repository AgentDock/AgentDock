# Implementation Plan Template

You are a senior implementation and project management expert with over 15 years of experience turning approved strategies into executed results. You have led implementation efforts across technology, healthcare, financial services, government, and manufacturing sectors. Your plans are known for translating high-level goals into specific daily and weekly actions that teams can follow without confusion. You understand that most initiatives fail not because the strategy was wrong but because the implementation lacked clarity, accountability, or realistic sequencing.

I need you to create a detailed implementation plan for the following initiative.

The project is called [PROJECT_NAME] and here is a description of the initiative:

[INITIATIVE_DESCRIPTION]

The primary objectives this implementation must achieve are:

[OBJECTIVES]

The scope of work covered by this plan is:

[SCOPE]

The implementation falls within the [DOMAIN:select:Software Deployment,Process Improvement,Organizational Change,Product Launch,System Migration,Policy Rollout,Digital Transformation,Infrastructure Build,Compliance Initiative,Training Program,Market Expansion,Merger Integration] domain.

The target timeline for completion is [TIMELINE:select:2 weeks,1 month,2 months,3 months,6 months,9 months,12 months,18 months,24 months]. The approved budget is [BUDGET:select:Under $10K,$10K-$50K,$50K-$100K,$100K-$250K,$250K-$500K,$500K-$1M,Over $1M,Not yet determined].

The available resources, including team members, tools, and systems, are:

[RESOURCES]

The key stakeholders and their roles or expectations are:

[STAKEHOLDERS]

Known dependencies that could affect sequencing or timing include:

[DEPENDENCIES?]

Risks I have already identified or that should be considered:

[RISKS?]

The criteria that will determine whether this implementation is successful:

[SUCCESS_CRITERIA]

Any additional constraints, organizational context, or priorities I should mention:

[ADDITIONAL_CONTEXT?]

Create a complete implementation plan document with the following sections:

1. Executive Summary - Summarize the initiative in three to four sentences covering what is being implemented, why it matters, the timeline, and the primary success measure. Write for a senior leader who reads nothing else.

2. Goals and Objectives - List each objective with a target metric, baseline value if available, and completion date. Connect each objective to the broader strategy that justified this initiative.

3. Scope Definition - State what is included and what is explicitly excluded. Clear boundaries prevent scope creep and protect the team from unplanned work.

4. Implementation Phases and Task Breakdown - Divide the work into three to six phases. For each phase, create a task table with columns for Task ID, Task Description, Owner, Duration, Start Date, End Date, Dependencies, and Status. Keep tasks small enough to complete in one to five business days. Include a brief narrative for each phase explaining its purpose.

5. Timeline and Milestones - Show each phase with start and end dates. Identify five to eight milestones with the deliverable that proves completion, the target date, and who approves it. Highlight the critical path and tasks with zero slack.

6. Resource Allocation - Map each resource to assigned tasks, hours per week, and periods of overallocation. Include personnel, budget by phase, tools, and technology. Flag resource gaps that must be resolved before launch.

7. Stakeholder Engagement Plan - For each stakeholder group, define their role, decisions they own, update frequency, and format. Include an escalation path for stalled decisions.

8. Risk Management Plan - List the top eight to ten risks with description, probability, impact, mitigation strategy, contingency plan, and monitoring owner. Categorize by type: technical, organizational, resource, or external.

9. Communication Plan - Define cadence and channel for kickoff meetings, weekly status updates, milestone reviews, steering committee briefings, and escalations. Specify who sends, who receives, and where records are stored.

10. Change Management Approach - Cover training requirements, user adoption strategy, resistance management, and feedback collection. Specify when training happens relative to each phase and how adoption rates are measured.

11. Quality Assurance and Acceptance Criteria - Define how deliverables are reviewed, tested, and approved. Include review criteria, turnaround times, and what happens when standards are not met.

12. Success Metrics and Evaluation Framework - For each metric, specify the data source, measurement frequency, responsible person, baseline, target value, and target date. Include leading indicators for early progress and lagging indicators for final results.

13. Post-Implementation Review Plan - Describe how the implementation will be evaluated after completion, including lessons-learned sessions, participants, and how findings will be documented for future teams.

Write in clear, direct language. Use tables for task breakdowns, resource allocation, and the risk register. Bold critical dates and decision points. If the stated timeline seems too aggressive for the scope, flag it and recommend a more achievable schedule.
