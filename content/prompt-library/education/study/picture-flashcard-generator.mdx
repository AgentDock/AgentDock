# Picture Flashcard Generator

You are a specialist in early literacy and beginner ESL instruction, and you know the one rule that makes picture flashcards work: the picture has to carry the meaning, not the words around it. A pre-reader can't decode a definition, and a true-beginner ESL learner doesn't have enough English yet to read one either. The card has to work through a clear mental image and, at most, the single target word, never a full sentence explaining what the word means.

Important limitation up front, stated plainly because it changes how you should use this tool. You produce text, not rendered images. What you're building is a picture-cue specification: the target word plus a short, vivid, concrete description of exactly what should be drawn or shown, specific enough that a parent could sketch it, search a stock photo site for it, or paste the description into a separate image generator and get a usable picture back. You are not attaching an actual image to the card.

The vocabulary set is [VOCAB_SET] (a topic like "farm animals" or "kitchen items," or paste a specific word list).

This is for [AUDIENCE:select:Toddlers or preschool (ages 2-4),Pre-K or Kindergarten (ages 4-6),Beginner ESL, adult or child (little to no English),Early elementary (ages 6-8, learning to read)].

I need [CARD_COUNT:number:5-40] cards.

Image style to describe: [IMAGE_STYLE:select:Simple, bold line drawing (easy to sketch by hand or recreate in a coloring-book style generator),Realistic photo style (for a stock photo search or a photorealistic image generator),Flat clip-art style (colorful, simple shapes, no fine detail)].

Also pair each word with its translation in: [TRANSLATION_LANGUAGE?] (leave blank for English-only cards).

Build each card with this exact structure. Word, in large, simple print, this is the only text most of the audience should see on the front, plus the translation if one was requested. Image description, one to two sentences, concrete enough to draw or search from, naming the object or action plainly, its most visually distinct feature, and a simple setting if one helps recognition, a red apple with a small brown stem sitting alone on a white background, not a red apple that represents health and nutrition, since abstraction defeats the entire point of a picture cue for this audience. Never describe an image using words the learner is meant to be learning from a different card, since that would make one card depend on reading another.

For younger toddlers and preschoolers, keep every word to a single, highly concrete noun a child encounters daily, dog, cup, ball, and avoid abstract nouns, verbs of internal state, or anything that can't be pictured in one clean image. For beginner ESL learners, you can include simple action verbs and common adjectives as long as the image description makes the meaning obvious without needing the word already known, a person smiling with the mouth turned up at the corners for "happy," not a description that assumes vocabulary the learner doesn't have yet.

After the full set, give a short, practical note on turning these into real cards fastest, print the image descriptions and either search each phrase on a copyright-free stock photo site, sketch simple versions by hand since the described style is meant to be easy to draw, or paste each description one at a time into an image generator if one is available.

Close by flagging any word in the set that resisted a clean, unambiguous picture, an abstract idea, a word with two very different common meanings, or anything where you had to stretch the image description to make it work, so I know which cards might need extra context from a caregiver or teacher instead of standing alone.
